# 履修科目選択ナビゲーター Webアプリ 要求仕様書（最新版）

---

## 1. プロジェクト概要

大学新入生（1年生）を対象とし、迷わずに卒業要件（100単位）に向けた履修計画を立てられるように導くスマートフォンファーストのSPA。  
サイバーパンク調のUIデザインで、アドバイザー教員と対話するような「診断フェーズ」を経て、学生個人の志向に合わせた「おすすめ科目」を提案。「シミュレーションフェーズ」で実際の科目をゲーム感覚で選択し、各種グラフで学習バランスを可視化する。

---

## 2. 技術スタック

- **フロントエンド:** HTML5, CSS3, Vanilla JavaScript (ES6+)
- **グラフ描画ライブラリ:** Chart.js (CDN経由)
- **AIテキスト生成:** Google Gemini API (gemini-2.0-flash)
- **構成:** サーバーレス静的サイト。バックエンドは持たず、データはJSのJSON配列として保持。

---

全科目データは、CSVの各DP小項目（DP1-1〜DP4-3の全20項目）を個別のパラメータとして保持する。  
実データは同じディレクトリ内の `data.js` に記録し、これをデータベースに組み込む。

```javascript

  {
    id: "C001",
    name: "授業科目名称",
    year: 1,               // 配当年次
    category: "科目区分",
    credits: 0,            // 単位数
    dp: {
      "1-1": 0, "1-2": 0, "1-3": 0, "1-4": 0, "1-5": 0, "1-6": 0,
      "2-1": 0, "2-2": 0, "2-3": 0, "2-4": 0, "2-5": 0,

    fields: { performance: 0, create: 0, business: 0 },
    domains: { music: 0, entertainment: 0, stage: 0, it: 0 },

];
```

## 4. アプリケーションのフローと機能要件（実装済み）

### 4.1. 起動フェーズ

- 「履修科目選択アシスタントを起動する」ボタンを表示。
- クリックでアシスタント起動メッセージを表示し、診断フェーズへ遷移。

### 4.2. 診断フェーズ（3ステップ）

1ステップ1画面のUI。各問は**優先順位付きの複数選択式ボタン**。  
選択済み項目は黄色ハイライト、未選択は明るめのオレンジ/緑系背景で表示。選択順に「1位・2位…」と一覧表示する。

#### Q1. 興味のある分野（最大5つ、優先順位付き）

| キー | ラベル |
|------|--------|
| pops | ポピュラー音楽 |
| classic | クラシック音楽 |
| sound | サウンドエンジニアリング |
| dance | ダンス・身体表現 |
| act | 演劇・声優 |
| vocal | オペラ・ミュージカル・声楽 |
| entertainment | エンタメ・総合芸術 |
| it | IT・先端技術 |

#### Q2. 興味のあるキーワード群（最大3つ、優先順位付き）

| キー | ラベル |
|------|--------|
| create | クリエイト・創作・制作・ものづくり |
| performance | パフォーマンス・演奏・表現・舞台に立つ |
| business | ビジネス・販売・チームワーク・誰かを支える |

#### Q3. 特に身につけたい力（最大5つ、優先順位付き）

DP全20項目からボタン選択。選択順に優先順位を表示。

#### スコアリングロジック

選択数に応じた重み付けを行い、科目ごとの適合度（おすすめ度%）を算出する。

| 選択数 | 重み配分 |
|--------|----------|
| 1 | 100% |
| 2 | 60 : 40 |
| 3 | 50 : 30 : 20 |
| 4 | 40 : 30 : 20 : 10 |
| 5 | 30 : 25 : 20 : 15 : 10 |

- **Q1：** 選択した分野に対応する `domains` が一致する科目に重み付き加点
- **Q2：** 選択したフィールドに対応する `fields` が一致する科目に重み付き加点
- **Q3：** 選択したDPに対応する `dp` 値に重み付き加点

### 4.3. メイン画面（診断完了後）

診断完了後、以下のセクションが順に表示される。

#### 参考資料ボタン

- 「参考資料：履修モデル一覧」ボタンを最上部に設置。
- クリックでGoogle Drive（履修モデルフォルダ）を別ウィンドウで開く。

#### おすすめランキング

- おすすめ度の高い科目上位10件をランキング形式で表示。
- 各科目の名称・科目区分・おすすめ度%を表示。

#### 履修シミュレーション

- 科目区分ごとにアコーディオン形式でまとめて表示。
- 各科目にチェックボックスと「おすすめ/必修」バッジを表示。
- 科目の選択・解除に連動してグラフがリアルタイムで再描画される。
- 必修科目（14科目）は常に選択状態に固定。
- 「おすすめ科目自動選択」ボタンで100単位になるよう自動選択。

#### 履修バランス可視化

- **DPレーダーチャート（20軸）：** Chart.jsによるレーダーチャート。各軸の色はDP区分ごとに色分け（DP1:青, DP2:緑, DP3:黄, DP4:赤）。Q3で選択したDPは白でハイライト。
- **DP大項目アーク（弧）：** レーダーチャート上に重ねて描画。DP1〜DP4の各大項目のDP統合値に応じた半径の弧を、各DP区分の色で表示。弧の角度はDP1:6/20・DP2:5/20・DP3:6/20・DP4:3/20で固定。透明度50%。
- **レーダーチャート四隅ラベル：** 右上「学科固有の能力（青）」・右下「知識・技能（緑）」・左下「汎用的能力（黄）」・左上「態度・姿勢（赤）」で大項目を明示。
- **円グラフ（2種）：** フィールド割合とドメイン割合をそれぞれ円グラフで表示。
- **合計単位数表示：** 選択中の合計単位数をリアルタイムで表示。

#### AI分析

- 「AI分析スタート」ボタンで、選択した履修科目・DP値・Q1〜Q3の回答をもとにGoogle Gemini APIを呼び出し、テキスト分析結果を表示。
- 分析結果は左揃えの白文字で表示。

#### 選択科目一覧

- 選択した科目を配当年次ごとにグループ化して表示。
- 年次ごとの上限は36単位。超過分は自動的に次年次へ移動。
- 左揃えで表示。

# 統合型・学修サイクルシステムの構想案

## 〜履修ナビゲーションWEBアプリを起点とした自律的学習支援基盤の実現〜

---

## 構想の背景と目的

- **現状の課題：** 学生の履修時における「何を学ぶべきか分からない」という初期の躓きや、システム間（LMSや教務システム等）で分断された学習記録の散逸。
- **目的：** 教務（教学デザイン）と情報（データ連携基盤）が協働し、学生が自律的に「学びのサイクル」を回せる統合型システムを構築する。
- **アプリの役割：** 卒業要件という明確なゴールを持つ正課カリキュラムに対し、予測の可視化・言語化によって確実性の高い「計画立案」を支援する。各カリキュラムの内容と獲得できるDP（能力）との因果関係を明示することで、透明性と信頼性の高い学修システムを構築し、学生の納得感と学修への主体性を強力に引き出す。

### アプリの挙動

1. 対話型UIによる興味関心・優先DPを選択肢、科目ごとに適合度をスコアリングする。
2. おすすめランキングと各科目ごとの必修・おすすめ表示に基づき履修科目を選択。
3. DPおよび科目区分傾向の可視化とAIによる言語化により、履修計画を具体化する。

---

## コア・コンセプト：正課カリキュラムを基盤とした中長期的な「学びのサイクル」

単一の枠組みによる単なる成績管理ではなく、大学のDP（ディプロマ・ポリシー）に基づく体系的かつ計画的な学修サイクルをシステム上で統合・支援する。

- **体系的な学修計画サイクル：** 卒業要件という明確なゴールに向け、中長期的な視点で普遍的な知の体系や専門性を確実に身につけるためのサイクル（カリキュラム中心／学期ごと）。

---

## 統合学修サイクルの4フェーズ

| フェーズ | 内容 |
|----------|------|
| **【計画立案】理想の自己デザイン** | WEBアプリを用い、100単位分のモデルプラン生成とDP獲得見込みを可視化。AIが強みや適職像を即座に言語化しフィードバックする。 |
| **【実行・蓄積】学修ポートフォリオのハブ化** | 立案した正課の学修計画の実行に加え、自己評価や振り返りなどの定量的・定性的な学習ログを一元的に集約する。 |
| **【評価・省察】メタ認知の促進** | 計画時に描かれた「理想のDP獲得見込み」と、実際の「現実のDP獲得状況」を比較・ギャップ分析し、自己省察を促す。 |
| **【対話・修正】データに基づく軌道修正** | 教員はAIアドバイスや達成水準の客観的データを基に面談を実施。日々の学習状況のデータを掛け合わせ、次期の履修計画へと繋げる。 |

---

## 履修シミュレーションシステムの継続的アップデート（予定）

- おすすめ科目抽出アルゴリズムの精度向上。
- 直感的な科目選択から、シラバスを必ず確認させた上で選択・決定させる教育的動線への変更。
- 選択肢から方向性を判断し、該当する領域を担当する教員名とゼミ（基礎演習）を表示する
- チュートリアル設置やアクセシビリティ向上を目的としたUI/UXの改善。

---

## 今後の課題（システム実装・運用に向けて）

- **機動的な学びとの段階的連携：** 社会変化や正課外での経験（インターンや課外活動等）に即応して柔軟に計画を見直す「機動的な学びのサイクル」については、初期構築フェーズでは基盤に組み込まず、体系的な学修サイクルが定着した後の次期拡張課題とする。将来的には両者を連携させる仕組みの確立を目指す。
- **DP評価指標の確立：** 「単位取得＝DP獲得」に留まらない、ルーブリックや自己評価を交えた客観的な評価エビデンスの要件定義。
- **教員の負担軽減：** 面談対象者のアラート抽出だけでなく、AIによる「声かけサジェスト」等を用い、教員の業務や認知負荷を下げる仕組みの導入。
- **段階的なデータ連携：** 既存システム群との完全連携を目指す前に、アプリ単体稼働からのスモールスタートとPoC（概念実証）ロードマップの策定。
- **生成AIの品質管理：** ハルシネーション（虚偽情報）を防ぐため、免責事項の明記や、大学推奨情報のみを参照させるRAG技術等を活用した責任分解の仕組み構築。
